## 一、绿武养成：限定副本 BOSS 掉材料 → 合成绿武

**你的意图**：绿武不再白送，改为"打限定副本 Boss → 掉绿武材料 → 合成/升阶绿武"。

**现状对照**
- ✅ 可复用：绿武实例、升级、进阶(阶段0-4已有 `weaponStages`/`stageRequirements`)、词条系统都现成。
- ❌ 缺口：没有"材料"数据结构，没有合成配方表，没有合成 UI。开局白送逻辑（第6712行 initialWeapon）与"合成获取"冲突。

**建议方案**
1. 新增**通用材料数据结构**（补现有短板）：`materials = { 玄铁精:{id,name,icon,tier}, 妖魂晶:{...}, ... }`，背包物品加 `category:'material'` 归类字段。
2. 新增**合成配方表** `weaponRecipes`：`{ 目标武器/阶段 : { 材料id:数量, ... , 需求等级 } }`。
3. **改造获取链**：保留开局白送"浅绿·青锋剑"作为新手过渡；**深绿武器改为合成产出**——限定副本 Boss 按配方掉材料，材料集齐在"合成台"合成。绿武进阶(1→4阶)也消耗对应材料，替代/补充现有纯几率进阶。
4. 新增**合成台 UI**（可挂在装备栏或新按钮），展示配方、材料库存、合成按钮。

**成本**：中偏高（新建材料+配方+UI三块）。**依赖**：第二点"限定副本"框架。

## 二、限定副本：每日次数限制 + 限定物品掉落

**你的意图**：限定副本每天有进入次数上限，控制产出通胀；掉专属限定物品（含绿武材料）。

**现状对照**
- ❌ 缺口：现在只有"世界地图"体系，没有独立的"副本"概念，没有每日重置逻辑。
- ✅ 可复用：`localStorage` 存档机制现成，`startBossBattle` 单场 Boss 战逻辑可复用为副本战斗核心，闲置的 `subMaps.bonus` 可作副本收益加成。

**建议方案**
1. 新增**限定副本数据表** `dailyDungeons`：`{ id, name, reqLevel, dailyLimit(如3次), enemies, bossId, dropTable(限定材料/物品) }`。
2. **每日次数**：存 `localStorage` 记录 `{dungeonId: {date:'2026-07-13', used:2}}`，进入时比对当天日期，跨天自动重置。次数耗尽则禁止进入并提示。
3. **入口**：地图面板新增"⚔️ 限定副本"分页，展示各副本剩余次数。
4. **限定掉落**：副本专属 `dropTable`，只有这里能掉绿武材料/高端物品（呼应第三点）。

**成本**：中。**依赖**：被第一、三点依赖，建议作为"产出经济"三件套(1+2+3)的地基先做。

## 三、高端物品只在副本 / 世界 BOSS 产出

**你的意图**：拉开产出档次——高端货只能打副本或世界 Boss 获得，普通刷怪只出低端。

**现状对照**
- ❌ 冲突：当前 `generateRandomEquip` 普通怪就能按 1% 掉神话品质，与你的意图直接矛盾，必须改。
- ✅ 可复用：`itemQualities` 六档品质表、`shouldDrop` 概率、`grantBossDrops` Boss 掉落框架现成。

**建议方案**
1. **给掉落函数加"来源"参数**：`generateRandomEquip(source)`，`source ∈ {'trash','dungeon','worldboss'}`。
2. **按来源限制品质上限**：
   - 普通刷怪(`trash`)：品质**封顶到"稀有"**（普通/精良/稀有），拿掉史诗+神话。
   - 限定副本(`dungeon`)：可出史诗，小概率传说。
   - 世界 Boss(`worldboss`)：可出传说/神话 + 限定绿武材料。
3. **补全装备数值**（顺带修空壳）：现在装备只有 `{type,quality}` 没属性。建议按 `equipTypes.stat × itemQualities.bonus × 等级` 生成实际属性，让掉落有意义。

**成本**：低-中（改掉落分级 + 补装备数值）。**依赖**：第二点副本框架。

## 四、换成大地图（山川河流废墟城阁，怪物在其中生成）

**你的意图**：把雷达同心圆战场，换成有地形（山川/河流/废墟/城阁）的大地图，怪物在地图里生成。

**现状对照**
- ⚠️ **这是 7 点里改动最大、风险最高的一项**。现在整套渲染/移动/刷怪都建立在"雷达同心圆 + 玩家永远居中 + 怪沿外圈刷"的假设上（`drawBackground`/`spawnEnemy`/点击移动/雷达半径 230）。换大地图 = 动战斗底层。

**建议方案（分期，强烈建议独立做）**
- **方案 A（轻量·推荐先做）**：保留雷达玩法内核，只做**视觉换皮**——`drawBackground` 按 `currentWorld` 主题绘制地形背景（山/水/废墟贴图或程序化色块），配合已有 `world.color`。低成本、高观感提升，不动战斗逻辑。
- **方案 B（重构·长线）**：真正的可探索大地图——引入世界坐标系 + 摄像机跟随 + 地形碰撞 + 分区刷怪。这等于重写渲染与移动核心，工作量最大，建议**单独立项、放在最后**，且先在 A 的基础上验证美术方向。

**成本**：A=低；B=高（近似重做战斗场景）。**依赖**：建议独立，不阻塞其它 6 点。

## 五、经验系统：约 90 天满级的长线 + 低级怪经验/掉落衰减

**你的意图**：设计长期玩法节奏（~90 天满级），并强化"打低级怪收益衰减"。

**现状对照**
- ✅ 已有：`expToNext ×1.2` 几何曲线、`getLevelPenaltyMult` 等级压制（越级碾压最低 0.05 倍收益）。
- ⚠️ 差距：现曲线到 99 级远达不到 90 天量级；衰减基于"**地图等级**"，不是"单只怪等级"（同图所有怪一个倍率）。

**建议方案（纯数值策划，成本最低，建议最先做）**
1. **重算经验曲线**：以"日均在线 X 分钟 × 日均可获经验"反推每级需求，把 `expToNext` 曲线拉成 90 天节奏（前期快、中后期显著放缓，典型分段：1-30 顺畅、30-60 平稳、60-90 长草、90-99 硬核）。
2. **补"单怪等级衰减"**：给 `enemyTypes` 或副本怪加 `level` 字段，`onEnemyKilled` 里按 `玩家等级 - 怪等级` 再叠一层个体衰减（现在只有地图级）。这样即使高级图里的小怪也会随你成长贬值，逼玩家往高图走。
3. 顺带修 Boss 战失败扣经验的字段 Bug（第二节⚠️2）。

**成本**：低（改公式+加字段）。**依赖**：无，可独立先行。

## 六、击飞连击技能系统 + 技能升级系统

**你的意图**：连招技能可以升级成长（不再固定数值）。

**现状对照**
- ✅ 已有：8 技能四段式闭环、连招编辑器、韧性/浮空系统都完整。
- ❌ 缺口：`comboSkills` 数值全写死，无 level/成长；但**闲置的 `skillPoints` 正好可当升级货币**。

**建议方案**
1. 给每个连招技能加**运行时等级层**：`comboSkillLevels = { zhongZhan:1, ... }`（存档），实际数值 = 基础值 × 等级成长系数（如每级 damageMult +8%、cost 微降、高级解锁额外 hits）。
2. **消耗 `skillPoints` 升级**：在连招编辑器里每个技能加"⬆️升级(消耗X技能点)"按钮，读现有 skillPoints。
3. 数值接入 `executeComboSkill` 现有计算链（把常量替换为"基础×等级系数"）。

**成本**：中（加成长层+UI+存档）。**依赖**：与第七点**共用 `skillPoints`，需先定分配规则**（见下方冲突提示）。

## 七、剑客天赋

**你的意图**：给剑客加天赋（成长/流派向的强化）。

**现状对照**
- ❌ 缺口：天赋系统完全不存在（`talent/天赋` 零匹配）。`classes` 只有剑客，`growth` 字段闲置，`stableStrike/evasiveBlock` 两条被动没落地。
- ✅ 可复用：`calculatePlayerStats` 的加成链是天赋接入的天然位置；`skillPoints` 可作天赋货币。

**建议方案**
1. 新增**天赋树数据** `talentTree`：分 3 路线（如"力斩/暴击输出"、"浮空连击"、"生存续航"），每个天赋节点 `{id, name, desc, maxRank, effect, cost, requires}`。
2. **消耗天赋点点亮**，效果注入 `calculatePlayerStats`（暴击/攻/减伤等）或连招系统（浮空时长、连招增伤）。
3. 新增**天赋面板 UI**（新按钮/快捷键 T）。
4. 顺带补全 `stableStrike`/`evasiveBlock` 两条被动，让职业设定完整。

**成本**：中-高（数据+效果注入+UI）。**依赖**：与第六点的货币分配。

> **⚠️ 关键冲突提示（6 与 7）**：目前只有一种 `skillPoints`，而"技能升级"和"天赋"都想用它。二选一：
> - **方案①（推荐）**：拆成两种货币——`skillPoints`(技能升级) + 新增 `talentPoints`(天赋，可设每 5 级给 1 点)，各管一摊，成长感更清晰。
> - **方案②**：共用一池，但要平衡"点技能还是点天赋"的取舍成本。

## 八、综合建议：优先级 · 依赖 · 实施路线

按"投入产出比 + 依赖顺序"排出建议节奏（每一阶段做完可独立体验、可提交）：

| 阶段 | 内容 | 理由 | 成本 |
|---|---|---|---|
| **P0 地基** | ⑤经验曲线重算 + 单怪衰减 + 修 3 个已有 Bug + 补普通装备数值 | 纯数值/修复，成本最低，是长线体验的根 | 低 |
| **P1 产出经济** | ②限定副本框架(每日次数) → ③掉落分级(高端限副本/世界Boss) → ①绿武材料合成 | 三者强关联，②是①③的地基，一条线做完形成完整产出循环 | 中-高 |
| **P2 养成深度** | 先定⑥⑦货币规则 → ⑥连招技能升级 → ⑦剑客天赋 | 复用闲置 skillPoints，给长线提供成长目标 | 中-高 |
| **P3 视觉/场景** | ④-A 大地图换皮(地形背景) →（远期评估）④-B 可探索大地图重构 | 风险最高、最独立，放最后；先低成本换皮验证方向 | A低 / B高 |

**核心取舍点（需你拍板）**
1. 第④点大地图：先做**低成本换皮(A)**，还是直接立项**重构可探索地图(B)**？（B 会大幅拖慢其它功能）
2. 第⑥⑦点货币：**拆成技能点+天赋点两种**，还是**共用一池**？
3. 绿武获取：开局**白送浅绿新手武器**、深绿改**合成**，这个过渡是否 OK？
4. 是否本轮先把 P0（数值+修 Bug）落地，让我先做出来给你看效果，再推进 P1？
